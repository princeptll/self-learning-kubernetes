## What are Labels in Kubernetes?

Labels are **key-value pairs** attached to Kubernetes objects like Pods, Services, Deployments, Nodes, etc.

They are used to **identify, organize, and select** resources.

### Example Labels

    labels:
      app: frontend
      env: production
      tier: web


Why Labels Are Important

Labels help Kubernetes to:

    Group related resources

    Select specific Pods for Services

    Filter resources

    Apply rules (scaling, networking, policies)

    Manage large clusters easily

Where Labels Are Used

Labels can be added to:

    Pods

    Deployments

    ReplicaSets

    Services

    Nodes

    ConfigMaps

    Secrets

## How to Add Labels

### 1. While Creating a Pod

    apiVersion: v1
    kind: Pod
    metadata:
      name: my-pod
      labels:
        app: myapp
        env: dev
    spec:
      containers:
        - name: nginx
          image: nginx

### 2. Using kubectl Command

    kubectl label pod my-pod app=myapp

#### Update an Existing Label

    kubectl label pod my-pod env=prod --overwrite

#### How to View Labels

    kubectl get pods --show-labels

Or:

    kubectl describe pod my-pod


#### What is a Label Selector?

Label selectors are used to find matching resources using labels.

#### Example

    kubectl get pods -l app=myapp


### How Labels Integrate with Kubernetes Components
1. Labels + Service

A Service uses labels to know which Pods it should send traffic to.

Pod Label:

labels:
  app: myapp

Service Selector:

selector:
  app: myapp

 2. Labels + Deployment

Deployments use labels to manage Pods.

    selector:
      matchLabels:
        app: myapp

 3. Labels + Node Scheduling

Assign labels to nodes:

    kubectl label node node1 disktype=ssd

Schedule Pods on that node:

    nodeSelector:
      disktype: ssd

4. Labels + Monitoring & Logging

Monitoring tools like Prometheus use labels to:

    Filter metrics

    Create dashboards

    Set alerts
